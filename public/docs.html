Headless CMS API

API REST para gestión dinámica de contenido

http://localhost:3000/api

📦 Recursos disponibles

RecursoEndpoint
Content Types/content-types
Fields/fields
Content (entries)/:contentType

🧩 1. Content Types

Definen los modelos (ej: posts, products, categories)

POST /content-types
Crear un nuevo content type
{
  "name": "posts",
  "displayName": "Posts",
  "description": "Blog posts",
  "isSingle": false
}
GET /content-types
Obtener todos los content types
GET /content-types/:id
Obtener un content type específico
DELETE /content-types/:id
Eliminar un content type

🧩 2. Fields

Definen la estructura de cada content type

POST /fields
Crear un campo
{
  "contentTypeId": "UUID",
  "name": "title",
  "displayName": "Título",
  "fieldType": "string",
  "isRequired": true,
  "position": 1
}
GET /fields/content-type/:contentTypeId
Obtener campos de un content type
PATCH /fields/:id
Actualizar un campo
DELETE /fields/:id
Eliminar un campo

🧠 Tipos de campos soportados

string text richtext number boolean date image relation

🧩 3. Content (Entries)

Contenido dinámico basado en contentType

POST /:contentType
Crear una entrada (ej: POST /posts)
{
  "title": "Hola mundo",
  "content": "Mi primer post"
}
GET /:contentType
Obtener todas las entradas
GET /:contentType/:id
Obtener una entrada específica
PATCH /:contentType/:id
Actualizar una entrada (parcial)
{"title": "Nuevo título"}
DELETE /:contentType/:id
Eliminar una entrada (soft delete)

🔎 Filtros dinámicos

GET /posts?title=hola
Filtrar por campo
GET /posts?status=published
Filtrar por estado

🧠 Flujo básico

1
Crear content type
2
Crear fields
3
Crear contenido
4
Consultar contenido

🚀 Ejemplo con curl

curl -X POST http://localhost:3000/api/posts \ -H "Content-Type: application/json" \ -d '{"title":"Hola","content":"Texto"}'

Todo se define en la base de datos - Construye un CMS completamente dinámico
